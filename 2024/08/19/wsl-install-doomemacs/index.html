参阅doomemacs的安装教程与安装过程中的提示信息就可以解决一些问题。下面是遇到的一些场外问题。

1. 安装较新版本的Emacs

Ubuntu官方源不提供较新版本(28.x, 29.x)的Emacs，需要另寻他路。最简单的方法是用snap：

但snap需要运行snapd服务，这需要systemd或其替代。WSL只在近期才引入了systemd，可能需要手动打开。

2. git的证书问题

具体原因不明，总之在git clone的时候会出现server certificate verification failed. CAfile: none CRLfile: none的错误。包括在doom install的时候。

解决方法是关掉验证：

  1. 设置环境变量GIT_SSL_NO_VERIFY=true
  2. git config http.sslVerify "false"
